# Протокол общения с модулем UWB650

Модуль UWB650 управляется по UART через AT-подобные команды. Документация основана на исходном коде драйвера и даташитах (V2.2, V11) в `docs/`.

## Физическое подключение

- **Интерфейс:** UART (8N1, без аппаратного управления потоком)
- **Скорость по умолчанию:** 115200 бод
- **Пины ESP32-S3:** TX=GPIO17, RX=GPIO18 (UART2)
- **Буфер:** 1024 байт

## Формат команд

### Отправка (ESP32 → UWB650)

```
UWBRFAT+{COMMAND}\r\n
```

Примеры:
```
UWBRFAT+POWER=8\r\n          # Установить мощность
UWBRFAT+POWER?\r\n           # Запросить текущее значение
UWBRFAT+RANGING=1,0000,0001\r\n  # Одиночный рейнджинг
UWBRFAT+RESET\r\n            # Перезагрузка модуля
```

### Ответ (UWB650 → ESP32)

Стандартный ответ на set-команду:
```
OK\r\n
```
или
```
ERROR\r\n
```

Ответ на query-команду:
```
{value}\r\n
OK\r\n
```

Ответ на рейнджинг:
```
+RANGING=({distance}),({rssi})\r\n
OK\r\n
```

Специальное сообщение при старте модуля:
```
Finished Startup\r\n
```

## Команды конфигурации

Все команды ниже реализованы в `uwb650_driver.c` через функции `uwb650_set_*()`.

### BAUDRATE — Скорость UART

```
UWBRFAT+BAUDRATE={code}\r\n
```

| Код | Скорость |
|---|---|
| 0 | 230400 бод |
| 1 | 115200 бод (по умолчанию) |
| 2 | 57600 бод |
| 3 | 38400 бод |
| 4 | 19200 бод |
| 5 | 9600 бод |

**Внимание:** Смена baudrate в `uwb650_apply_config()` пропускается — это разорвало бы связь с ESP32.

### DATARATE — Скорость передачи данных UWB

```
UWBRFAT+DATARATE={code}\r\n
```

| Код | Скорость |
|---|---|
| 0 | 850 Kbps |
| 1 | 6.8 Mbps (по умолчанию) |

### DEVICEID — PAN ID и адрес узла

```
UWBRFAT+DEVICEID={PAN_HEX},{ADDR_HEX}\r\n
```

Пример: `UWBRFAT+DEVICEID=0000,0001\r\n`

- PAN ID: 0x0000–0xFFFE
- Адрес: 0x0000–0xFFFE
- По умолчанию: PAN=0x0000, Addr=0x0000

### POWER — Мощность передатчика

```
UWBRFAT+POWER={level}\r\n
```

| Уровень | dBm |
|---|---|
| 0 | -5.0 |
| 1 | -2.5 |
| 2 | 0.0 |
| 3 | 2.5 |
| 4 | 5.0 |
| 5 | 7.5 |
| 6 | 10.0 |
| 7 | 12.5 |
| 8 | 15.0 |
| 9 | 20.0 |
| 10 | 27.7 (по умолчанию, максимум) |

### PREAMBLECODE — Код преамбулы

```
UWBRFAT+PREAMBLECODE={code}\r\n
```

Диапазон: 9–24. По умолчанию: 9.

Оба устройства (инициатор и ответчик) должны использовать одинаковый код преамбулы.

### CCAENABLE — Clear Channel Assessment

```
UWBRFAT+CCAENABLE={0|1}\r\n
```

По умолчанию: 0 (выключено).

### ACKENABLE — Подтверждение приёма

```
UWBRFAT+ACKENABLE={0|1}\r\n
```

По умолчанию: 0 (выключено).

### SECURITY — AES-128 шифрование

```
UWBRFAT+SECURITY={0|1}[,{hex_key}]\r\n
```

- `SECURITY=0` — выключить
- `SECURITY=1,00112233445566778899AABBCCDDEEFF` — включить с ключом (32 hex-символа)

По умолчанию: выключено.

### TXTARGET — Цель для передачи

```
UWBRFAT+TXTARGET={PAN_HEX},{ADDR_HEX}\r\n
```

Пример: `UWBRFAT+TXTARGET=0000,0001\r\n`

### RXSHOWSRC — Показ адреса источника при приёме

```
UWBRFAT+RXSHOWSRC={0|1}\r\n
```

По умолчанию: 1 (включено).

### LEDSTATUS — Управление LED

```
UWBRFAT+LEDSTATUS={0|1}\r\n
```

По умолчанию: 1 (включено).

### RXENABLE — Включение приёмника

```
UWBRFAT+RXENABLE={0|1}\r\n
```

По умолчанию: 1 (включено).

### SNIFFEN — Режим прослушивания

```
UWBRFAT+SNIFFEN={0|1}\r\n
```

По умолчанию: 0 (выключено).

### ANTDELAY — Задержка антенны

```
UWBRFAT+ANTDELAY={value}\r\n
```

Диапазон: 0–65535. По умолчанию: 16400.

Используется для калибровки точности измерения расстояния.

### DISTOFFSET — Коррекция расстояния

```
UWBRFAT+DISTOFFSET={cm}\r\n
```

Диапазон: -500..500 (сантиметры). По умолчанию: 0.

### COORDINATE — Координаты устройства

```
UWBRFAT+COORDINATE={x},{y},{z}\r\n
```

Значения в сантиметрах. По умолчанию: 0,0,0.

## Команды управления

### FLASH — Сохранение в Flash модуля

```
UWBRFAT+FLASH\r\n
```

Сохраняет текущую конфигурацию модуля в его энергонезависимую память.

### RESET — Перезагрузка модуля

```
UWBRFAT+RESET\r\n
```

Модуль перезагружается. После загрузки отправляет `Finished Startup`. Драйвер ожидает это сообщение до 3 секунд.

### DEFAULT — Сброс к заводским настройкам

```
UWBRFAT+DEFAULT\r\n
```

Сбрасывает все параметры модуля и перезагружает его.

### VERSION — Версия прошивки модуля

```
UWBRFAT+VERSION?\r\n
```

Ответ: строка с версией, затем OK.

## Запросы (Query)

Любой параметр можно запросить, добавив `?` вместо `=value`:

```
UWBRFAT+POWER?\r\n
→ 10\r\n
→ OK\r\n

UWBRFAT+DEVICEID?\r\n
→ 0000,0001\r\n
→ OK\r\n
```

## Рейнджинг

### Одиночное измерение

```
UWBRFAT+RANGING=1,{TARGET_PAN_HEX},{TARGET_ADDR_HEX}\r\n
```

Пример: `UWBRFAT+RANGING=1,0000,0001\r\n`

**Ответ (успех):**
```
+RANGING=(3.46),(-67.50)\r\n
OK\r\n
```

**Ответ (неудача — цель не отвечает):**
```
+RANGING=(-1),(0.00)\r\n
OK\r\n
```

### Формат ответа рейнджинга

```
+RANGING=({distance_m}),({rssi_dBm})
```

- `distance_m` — расстояние в метрах (float). `-1` при ошибке
- `rssi_dBm` — уровень сигнала в dBm (float). `0.00` при ошибке

### Непрерывный рейнджинг (реализация драйвера)

Непрерывный рейнджинг реализован на стороне ESP32, а не модуля. Драйвер создаёт FreeRTOS-задачу, которая в цикле с частотой 10 Hz вызывает одиночный рейнджинг.

## Таймауты

| Операция | Таймаут | Константа |
|---|---|---|
| AT-команда (OK/ERROR) | 1000 мс | `UWB650_CMD_TIMEOUT_MS` |
| Рейнджинг (+RANGING=) | 2000 мс | `UWB650_RANGE_TIMEOUT_MS` |
| Ожидание Finished Startup | 3000 мс | `UWB650_STARTUP_WAIT_MS` |
| Пауза между командами | 50 мс | `CMD_DELAY_MS` |

## Порядок применения конфигурации

Функция `uwb650_apply_config()` отправляет команды в следующем порядке:

1. RESET (перезагрузка модуля)
2. DATARATE
3. DEVICEID
4. POWER
5. PREAMBLECODE
6. CCAENABLE
7. ACKENABLE
8. SECURITY
9. TXTARGET
10. RXSHOWSRC
11. LEDSTATUS
12. RXENABLE
13. SNIFFEN
14. ANTDELAY
15. DISTOFFSET
16. COORDINATE
17. FLASH (сохранение в модуль)

**Примечание:** BAUDRATE намеренно пропускается — смена скорости UART требует перенастройки ESP32, что может привести к потере связи.
